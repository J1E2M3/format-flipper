# format-flipper

Convert between **JSON, NDJSON, YAML, TOML, INI, .properties, CSV, TSV, XML, Markdown tables, HTML tables, and SQL INSERT** — 12 formats, any direction — as a Node library and CLI.

This is the exact engine behind the [Format Flipper browser tool](https://toolymctoolface.com/format/): `engine.js` is generated byte-for-byte from the tool's source, so the library converts exactly like the web page does. Your data is processed on your own machine — same zero-upload property as the browser tool, different execution context.

## Install

```bash
npm install format-flipper        # library
npm install -g format-flipper     # CLI
```

## CLI

```bash
# stdin to stdout
cat users.json | format-flipper -f json -t csv

# formats inferred from extensions
format-flipper -t yaml config.toml -o config.yaml

# batch: every CSV in data/ becomes JSON in out/
format-flipper -t json --out-dir out/ data/*.csv

# options
format-flipper -f csv -t sql --sql-table users --sql-quote ansi --strict < users.csv
```

Run `format-flipper --help` for the full flag list (`--indent`, `--strict`, `--no-coerce`, `--delimiter`, `--sql-table`, `--sql-quote`).

## Library

```js
const ff = require('format-flipper');

ff.convert('a: 1\nb: two', 'yaml', 'json');
// -> '{\n  "a": 1,\n  "b": "two"\n}'

ff.parse('id,name\n1,Ada', 'csv');
// -> [{ id: 1, name: 'Ada' }]

ff.serialize([{ id: 1 }], 'sql', { sqlTable: 'users' });
// -> CREATE TABLE `users` ... INSERT INTO `users` ...

ff.FORMATS;               // ['json', 'ndjson', 'yaml', ...]
ff.formatForExtension('yml'); // 'yaml'
```

Options accepted by `convert`/`parse`/`serialize` mirror the browser tool's options panel: `jsonIndent`, `yamlIndent`, `xmlIndent`, `xmlRoot`, `xmlItem`, `sqlTable`, `sqlQuote` (`backtick`|`ansi`|`bracket`), `coerceTypes`, `strictParse`, `csvDelimiterIn`, `csvDelimiterOut` (`comma`|`semicolon`|`pipe`|`tab`).

## How lossiness works

The intermediate representation is plain JavaScript values, which makes most conversions exact and a few predictably lossy (comments drop, TOML dates become ISO strings, integers past 2^53 lose precision). The [architecture notes](https://github.com/J1E2M3/format-flipper/blob/main/ARCHITECTURE.md) document every case.
